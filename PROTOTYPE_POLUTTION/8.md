# Lab: Detecting server-side prototype pollution without polluted property reflection

Challenge Description:
This lab is built on Node.js and the Express framework. It is vulnerable to server-side prototype pollution because it unsafely merges user-controllable input into a server-side JavaScript object.

To solve the lab, confirm the vulnerability by polluting Object.prototype in a way that triggers a noticeable but non-destructive change in the server's behavior. As this lab is designed to help you practice non-destructive detection techniques, you don't need to progress to exploitation.

You can log in to your own account with the following credentials: wiener:peter 

Goal:
 Detect and exploit prototype pollution even when the polluted property is not reflected in the response.

Solution:
 English: Set __proto__.status to 503 to pollute the server-side prototype without visible reflection.
 Indonesian: Atur __proto__.status ke 503 untuk mencemari prototype sisi server tanpa memantulkan properti secara langsung.

Step:
1. Pertama sesuai deskripsi chall cari prototype pollution source seperti `__proto__`
2. Selanjutnya cari pemicu dimana ini bisa dijalankan yaitu di `POST /my-account/change-address` jika di capture dengan burpsuite dan send request, response nya akan menunjukkan jika tidak reflected tapi merespon dengan 500 error response code dan status property nya dengan value 400
3. coba inject dengan `"__proto__":{ "status":503}`

Payload:
"__proto__":{ "status":503}
