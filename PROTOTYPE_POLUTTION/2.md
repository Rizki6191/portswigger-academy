# Lab: DOM XSS via client-side prototype pollution

Challenge Description:
 This lab is vulnerable to DOM XSS via client-side prototype pollution. To solve the lab, find a source that can add arbitrary properties to the global Object.prototype, identify a gadget property that allows execution of arbitrary JavaScript, and combine these to call alert().

Goal:
 Use a prototype pollution source to change a gadget property and execute alert() from the browser.

Solution:
 English: Start with the __proto__[transport_url] parameter and point it at data:,alert(1);.
 Indonesian: Mulai dengan parameter __proto__[transport_url] dan arahkan ke data:,alert(1);.

Step:
1. Pertama cari prototype pollution source seperti `__proto__` lalu juga cari gadgetnya, disini gadget nya adalah `transport_url`
2. Setelahnya coba dengan `?__proto__[transport_url]=foo` dan cek dengan `console.log(Object.prototype.transport_url);` jika menghasilkan output foo berarti prototype pollution berhasil
3. Karena tujuan nya adalah memicu alert() yang berarti harus mencari pemicunya, disini sama seperti sebelumnya bisa dengan format data URI `data:[<mediatype>][;base64],<data>`, dan ini payload akhirnya `/?__proto__[transport_url]=data:,alert(1);`

Payload:
/?__proto__[transport_url]=data:,alert(1);
